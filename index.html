Laravel app for job search.

The app is built using the Laravel framework. It utilizes the `ListingController` to render the index view and pass arguments from the database using the `Listing` model. With a helper method, the listing model is filtered to display the latest jobs. Additionally, the results are filtered when clicking on any tag. Another filter is the search input, where the value is sent by a GET request and compared against the table of title, description, and tags in the database. If there is a match, we get filtered jobs.

Every job has its own dynamic route created in the `web.php` file. Additionally, each route is protected with middleware, depending on whether the user is logged in or not. When a user is logged in, it is possible to create a job offer and also update or delete the same job offer.

Inside the `views/components` folder, there are Blade.php files that handle functionalities such as registration and login. The `components` folder contains files that I use more than once, such as a layout containing the header and footer. Inside the layout, there are deliveries that I use across the whole app, like Tailwind CSS and Alpine.js. Another component is `listing-card`, which contains all the information from the database for every job listed on the site.

For registration and sign-in, I use a `UserController` to validate fields inside the registration form and to compare email and password fields with the database in the login form.

Gym Panel App

To use the app, you need to have active credentials.

The Gym Panel App is a versatile PHP-based application designed primarily for gym administrators but with the flexibility to adapt to various business needs. Its main purpose is to empower gym administrators with a comprehensive set of tools for efficient management and administration.

When a user logs in with the correct credentials, it is possible to see the main menu that contains a list of members. It retrieves information from the database about each member info.The SQL query retrieves comprehensive information about members in a database by combining data from multiple tables. It establishes connections between the members table, training_plans table, and trainers table to assemble a detailed dataset. This allows for a holistic view of each member's details, including their training plan, assigned trainer, and relevant trainer information.

Register member and trainer handle form submission via POST method, capturing member and trainer details. It then inserts the data into a MySQL database table named members and trainers. Additionally, it generates a PDF access card for the new member, incorporating information like member ID, name, email, and training plan details. The access card PDF is stored, and its file path is updated in the database. Finally, a success message is set, and the user is redirected to an admin dashboard for members.

Images are stored inside the 'members_photos' folder, and when the correct path is retrieved from the database, it shows the corresponding image. If the image table is empty, the main picture will show up for both members and trainers.

It is possible to assign a trainer from the list to any member, and that information will be updated inside the database. Training plans can be set up, deleted, or updated for existing plans.

E-commerce App

To use the app, you need to have active credentials, for admin and user panel.

E-commerce application, built using PHP and MySQL, offers seamless login and registration for users. Within the user panel, customers can easily select the desired quantity of products and add them to their shopping cart. Once users have chosen the products they wish to purchase, they can proceed to place an order. During the ordering process, users provide essential information such as address, city, and state. The order details are then securely stored in the database, effectively simulating order fulfillment process.

Admins log in using the same interface as regular users. Once logged in, admins can easily add new products with details like name, price, size, and images. They also have the ability to update or remove existing products, giving them control over the product catalog.

Social media App

Social network with the possibility to log in with an existing account, register a new account, post something on the wall, comment on every post individually. Users can edit their username and password, logout and delete their profile. Info about users, posts, and comments are stored in mockAPI database (https://mockapi.io/) using fetching.

This app is a good example of using OOP and asynchronous functions. I utilize API endpoints for storing, updating, and deleting inputs. By defining the method and specifying the file format in the header, I can send objects inside the body to the endpoint, store them, call them from the API, and present them when needed.

Weaher App

An app that uses your device location to show the weather in your city. displaying current temperature, and three separate areas that show: daily info about - high, low, max feels like, min feels like temperature, wind speed, and precipitation. weekly info about the sky icon, and max temperature for that day hourly info about temperature, feels like temp, wind speed, and precipitation.

I use the API https://api.open-meteo.com, and from there, I get all the information I need for the current weather. Inside the fetch, I dynamically set up longitude and latitude so that I get only weather info for the current location and the name of the city as well. Information about weather is set up in three categories: current, daily, and hourly. Each category has its own function which helps me render current and future information about all weather details and present it to the user.

Shopping Cart App

Building a shopping cart using local data in folder Data.js. building html structure adapting the site for different sizes of screens implementing functionality with Java Script

The app utilizes an array of objects locally set up to simulate a database. It leverages array methods such as filter, map, reduce, and find to implement various functions. These functions include increasing or decreasing product quantities, finding the right product using its ID, calculating the total sum of costs for all products, setting up products in local storage, and deleting from local storage if they are no longer in the cart.
